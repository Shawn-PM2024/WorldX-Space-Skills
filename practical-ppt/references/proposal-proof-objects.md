# Proposal Proof Objects

Use this reference when the deck is a joint venture, financing, strategy, board, shareholder, or executive decision proposal.

The goal is not to make pages decorative. The goal is to turn each required point into a visible proof object that can be understood from the contact sheet.

## Benchmark Gap Audit

When the user provides a competing deck, benchmark deck, or another tool's output for the same input, audit both before changing the skill output.

Compare at least:

- slide count and narrative spine
- title claim quality and slide jobs
- text density and minimum readable size
- proof object type per slide
- visual rhythm across cover, body, comparison, financial/decision, appendix, and ending pages
- media/native asset usage: dot grids, rules, dividers, badges, charts, icons, schematic shapes, product/market visuals
- repeated layout signatures: too many pages using the same card grid, table, or boxed-list grammar
- content compression: what was merged, split, summarized, or made visual

Write a `benchmark-gap-brief.md` with:

- what the benchmark does better
- what the current deck does better
- which gaps are style only
- which gaps are story/proof-object gaps
- exact slide-level changes to make before rebuilding

## Joint Venture / Executive Proposal Spine

A strong proposal deck usually needs this rhythm:

1. Cover: clear proposal name, parties, audience, date, and one premium visual motif.
2. Core conclusion: one-sentence decision frame plus 3-5 key facts.
3. Strategic rationale: why this entity/platform is needed now.
4. Company or shareholder role: each party's contribution and control logic.
5. Business scope: what the JV owns, sells, builds, and excludes.
6. Commercial path: first customer, repeatable market, expansion logic.
7. IP and asset boundary: what is transferred, licensed, retained, and protected.
8. Cap table and governance: ownership, contribution, board/control, incentives.
9. Option comparison: recommended plan versus alternatives.
10. Revenue and cash return: how money flows to each party.
11. Long-term operating model: who drives demand, product, R&D, sales, and platform assets.
12. Team and milestone plan: phased team build, revenue triggers, cost control.
13. Legal documents and next steps: agreements, owners, timing.
14. Shareholder talking points / appendix.
15. Ending page: decision asks and next action.

Do not force this exact order when the user's outline says otherwise. Use it as a coverage checklist.

## Proof Object Toolbox

Choose one primary proof object per substantial slide.

| Slide job | Strong proof object | Avoid |
| --- | --- | --- |
| Core conclusion | verdict strip + 3 KPI/stat cards + operating flow | paragraph-only executive summary |
| Company/shareholder role | contribution map + product/market cards | generic "about us" bullet list |
| Business scope | value chain or capability stack | four equal cards with long bullets |
| Commercial path | 3-stage funnel or market expansion ladder | plain roadmap with tiny labels |
| IP boundary | quadrant, rights matrix, or ownership swimlane | dense legal table only |
| Cap table | donut/ring + contribution cards + governance strip | table only |
| Option comparison | 2-column decision table + recommendation badge | equal-weight comparison with no winner |
| Revenue logic | revenue stack, cash waterfall, or 1x vs 3-5x bridge | bullet list of fee types |
| Long-term cooperation | role swimlane or operating loop | five same-size boxes |
| Team plan | phased timeline with triggers | HR table only |
| Legal next steps | checklist table with owners and dates | paragraph of next steps |
| Talking points | objection/answer cards | unsorted appendix bullets |

## Visual Asset Grammar

Business decks need reusable native assets even when no external images are available:

- dot-grid motifs for technical/platform feel
- thin horizontal/vertical rules for section structure
- numbered badges, stage labels, and recommendation stamps
- subtle background blocks that do not reduce contrast
- donut/ring charts for ownership and mix
- arrow strips, chevrons, ladders, and funnels for process
- small icon-like marks built from native shapes when icons are unavailable

Prefer native editable shapes. Rasterized assets are acceptable only for motifs, screenshots, photos, or visual evidence. Core text and diagrams must stay editable.

## Density Rules

- Keep one slide claim and one proof object per slide.
- If a slide needs more than 550 Chinese characters or 8 text boxes, split it or convert part of it into a diagram.
- If any text would need to drop below 12pt, split the slide, shorten the copy, or move details to appendix.
- If 3 consecutive slides use the same table/card layout, change one into a funnel, timeline, matrix, stack, ring, or swimlane.
- Contact sheet test: at thumbnail size, each body slide should still show a distinct object shape.
