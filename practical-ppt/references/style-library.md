# Practical PPT Style Library

This file stores reusable template types and visual grammar for practical PPT generation. Extend it by adding new template types and usage rules.

Template types are style grammar, not fixed layouts to clone. For every new deck, adapt the grammar to the actual topic, audience, and content density.

## Template Type Index

- **Minimal Pitch**: early-stage pitch style, white space, one accent color, sparse text, simple silhouettes, plain comparison tables.
- **Red Strategic Planning**: red-white planning style, bold chapter numbers, dense organization/process diagrams, circular models, strong executive hierarchy.
- **Dark AI Infrastructure**: dark technical style, cinematic contrast, low-key technical backgrounds, white text, timelines, architecture maps, team cards.
- **Blue-White Corporate Tech**: light technology company style, glass panels, geometric motifs, structured contents, company/profile/product sections.
- **Adaptive Business Modular**: topic-routed business template system, modular section rhythm, remixable layouts, strong narrative skeleton.
- **Editable Diagram Grammar**: native-shape diagram system for list, process, cycle, hierarchy, relationship, matrix, pyramid, and picture/timeline diagrams.

## Style Adaptation Rule

Before drawing slides, write a short style brief:

- **Topic fit**: what kind of deck this is, such as company intro, method training, product narrative, investor pitch, internal planning, or technical platform.
- **Audience fit**: who reads it and how much detail they need.
- **Template grammar**: which template family contributes layout, hierarchy, rhythm, or palette.
- **Deliberate modifications**: what changes make it feel native to this topic rather than copied from a template.
- **Do not copy**: concrete elements to avoid, such as exact arcs, exact cover composition, fake logos, repeated decorative chrome, or irrelevant image motifs.

Use the closest template family as a starting grammar, then tune palette, layout density, proof objects, and page rhythm until the deck feels purpose-built.

## External Template Routing

### Adaptive Business Modular

Use this type when the deck topic is more important than matching a fixed brand template.

Observed reusable rules:

- Route by use case first: company, consulting, creative, education, fundraising, marketing, people, project management, reporting, sales, and strategy.
- Treat templates as adaptable structures. Start from a proven narrative skeleton, then remix sections, density, visuals, and tone for the actual content.
- Use modular slide blocks: hero/overview, key takeaways, framework, process, comparison, evidence, case/example, risks, action plan, and close.
- Match each block to a proof object rather than a decoration: SWOT, market sizing, root-cause tree, strategic priorities, roadmap, retrospective, deal overview, onboarding path, report summary.
- Keep cards responsive in spirit: avoid fixed page designs that collapse when text length changes; use grids, stackable sections, and cards that can be expanded or split.
- Prefer strong first screens, clear section transitions, and consistent rhythm over ornate single-slide effects.

Adaptive subtypes:

- **Clean Business Modular**: white/light neutral background, crisp cards, clear titles, restrained accent color. Use for reports, project updates, onboarding, and general business decks.
- **Dark Executive**: dark background, bold contrast, fewer words, big claim and proof object. Use for executive reviews, strategic pitches, and high-stakes decisions.
- **Warm Proposal**: warm neutrals, rounded panels, soft accent, proposal-like flow. Use for consulting, service proposals, grants, plans, and stakeholder narratives.
- **Analytical Framework**: dense but organized matrices, quadrant maps, root-cause trees, SWOT blocks, and priority grids. Use for analysis, strategy, decision, and diagnosis decks.
- **Creative Editorial**: strong imagery, editorial spacing, asymmetric composition. Use only when the topic is brand, creative, portfolio, culture, or story-led.

Do not copy an existing template literally. Copy the routing discipline: pick the template logic that fits the task, then make the visual system native to the user's topic.

### Editable Diagram Grammar

Use this type for editable diagram structures. Rebuild diagrams with native PowerPoint shapes, lines, arrows, and text so the user can modify them.

Choose diagram grammar by content relationship:

- **List**: horizontal bullet list, grouped list, alternating list, vertical box list. Use for non-sequential peer items, principles, feature groups, and checklists.
- **Process**: continuous arrow, chevrons, stepped process, descending steps, circular process with feedback. Use for workflows, procedures, timelines, and task sequences.
- **Cycle**: circular arrows, segmented cycle, hub-and-loop, feedback loop. Use for iterative systems, review loops, growth loops, and recurring operations.
- **Hierarchy**: org chart, tree, layered stack, branching structure. Use for teams, taxonomies, capability maps, architecture layers, and command structures.
- **Relationship**: gears, radial network, opposing arrows, Venn/overlap, hub-and-spoke. Use for cause-effect, interdependence, tradeoffs, stakeholder maps, and system interactions.
- **Matrix**: 2x2, quadrant, grid, labeled axes, central box with surrounding blocks. Use for prioritization, comparison, segmentation, and risk/opportunity maps.
- **Pyramid**: upright pyramid, inverted pyramid, segmented pyramid, pyramid plus callouts. Use for maturity models, priority stacks, hierarchy of needs, funnel-like narrowing, and capability levels.
- **Picture / Timeline**: circle-emphasis timeline, image-caption blocks, picture list, framed gallery. Use for case studies, milestones, product screenshots, people, and visual evidence.

Diagram adaptation rules:

- Keep the semantic structure, not the default gray look.
- Use the deck's palette, typography, and spacing.
- Label shapes with short text; move detail into side notes or supporting cards.
- For 5+ steps, prefer a timeline, chevron row, or vertical ladder over equal-width cards.
- For 6+ peer items, group into clusters or use a matrix instead of a flat card grid.
- For dense diagrams, split into two slides if labels drop below readable size.

### Blue-White Corporate Tech

Use for company introductions, product portfolios, solution decks, industry scenarios, and partner/ecosystem narratives.

- Background: light blue-white gradients, faint geometric panels, subtle grid or cube motifs.
- Palette: deep blue primary, cyan/sky secondary, white panels, slate text.
- Typography: strong title in 44-64px, section labels in 18-24px, body 24-32px, captions 16-18px.
- Layouts: contents page with numbered blocks, two-column profile pages, product cards, solution matrices, partner ecosystem maps.
- Details: small corner brand label, thin dividers, soft shadows, translucent panels, icon-led bullet groups.

### Red Strategic Planning

Use for annual summary, planning, roadmap, organization review, KPI progress, and internal executive reporting.

- Background: white or very light gray with red blocks/arcs, or red cover/transition pages.
- Palette: red primary, pale red fills, charcoal text, warm gray dividers.
- Typography: huge chapter numbers, compact Chinese labels, strong section headers.
- Layouts: department maps, yearly timeline, milestone lanes, ring diagrams, capability pyramids, issue/action tables.
- Details: use red for hierarchy and progress, not as a full-page wash except on cover or transition slides.

For writing, methodology, training, or creator-workflow topics, soften this family into `Editorial Method`. Keep red as an editorial mark and decision signal, reduce corporate planning chrome, add more whitespace, and use notebook-like panels, process ladders, and checklist modules.

### Dark AI Infrastructure

Use for AI infrastructure, model/chip/cloud platform, deep technology, financing, and high-stakes technical narratives.

- Background: black-purple or deep plum gradients with subtle technical imagery.
- Palette: near-black, purple, magenta, electric blue, white text.
- Typography: large white titles, restrained accent labels, small but legible technical captions.
- Layouts: architecture diagrams, milestone timelines, team/profile cards, market maps, capability clusters.
- Details: maintain contrast; use glow sparingly around central proof objects.

### Minimal Pitch

Use for founder-style pitch, concept explanation, early product logic, and short investor story.

- Background: mostly white.
- Palette: black/gray text with one red accent.
- Typography: simple sans serif, large titles, sparse body copy.
- Layouts: one idea per slide, sparse comparison tables, sequence of problem/solution/demo slides.
- Details: whitespace is the design system. Do not add decorative panels to fill empty space.

## Shared Layout Principles

- Use a clear top-left or center-title hierarchy. Do not split attention with multiple competing headlines.
- Vary rhythm across slides: cover, contents, section transition, proof slide, diagram slide, summary slide.
- Keep body text blocks short. Convert dense paragraphs into tables, diagrams, lanes, or callout groups.
- Use charts and diagrams as proof objects, not decoration.
- Keep page chrome consistent: logo/section marker, page number, and footer positions should not jump.
- Prefer 16:9 for new decks. Use 4:3 only when matching a 4:3 reference or user requirement.

## Anti-Patterns

- Single-color monotone decks with no hierarchy beyond font size.
- Generic card grids on every slide.
- Blindly copying another template's arcs, gradients, page chrome, or cover layout when the topic calls for a different metaphor.
- Unverified logos or pseudo-brand marks.
- Tiny dense text copied from the outline without slide rewriting.
- Decorative gradients that reduce readability.
